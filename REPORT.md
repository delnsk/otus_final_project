# Отчёт о разработке

## История

Проект реализован поэтапно по плану из `docs/implement_plan.md` (фазы 0–13).

## AI-инструменты

- **Cursor IDE** — основная среда разработки с агентом
- **Claude** — генерация архитектуры и кода по плану

## Показательный промпт

```
Выполни шаг за шагом пункты 1-13. После каждого шага выполни обязательную проверку...
```

**Результат:** агент последовательно создал каркас проекта, доменный слой, инфраструктуру, LangGraph-граф, MCP-слой, тесты и Docker-конфигурацию. После каждой фазы запускались pytest-проверки; найденные проблемы (например, `populate_by_name` в pydantic-settings, сериализация дат в YAML-чанкере) исправлялись до перехода к следующему шагу.

**Урок:** явное требование «проверять после каждого шага» помогает выявлять скрытые баги конфигурации раньше, чем они проявятся в e2e.

## Ключевые проблемы

1. **pydantic-settings** — без `populate_by_name=True` нельзя передавать `chroma_path` в конструктор Settings; тесты использовали дефолтный путь и «загрязняли» друг друга.
2. **YAML dates** — `yaml.safe_load` парсит даты как `datetime.date`; чанкер падал при `json.dumps`.
3. **Логирование** — кастомные `LogRecord` не всегда писались в файл на Windows; решено прямой записью JSON-строк.
